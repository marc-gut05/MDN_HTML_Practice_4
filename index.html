Citations

Citations are an important part of all information transfer. Having robust information displayed on your website is important, but allowing for there to be a clear path to the source that informed the interpretation is essential to creating trust and maintaining integrity. The need for citations is not only essential in collegiate writing, but in everyday life. Interpretations are important, but facts are paramount in all instances of media, literature, and conversation.

Unfortunately for readers of the website, browsers can not do much with included citations. Broswers do not display these citations, so it is important to incorporate these citations in a different fashion.

Instead of using the <cite> element to include a citation within the browser, we will use the <a> (anchor) element to create a hyperlink attached to a piece of text.

Below is an example of incorporating a link into a line of text.

According to the United States Census Bureau the United States population stands at just below 330 million people, with one birth occuring nearly every 8 seconds, one death occuring every 12 seconds, an addition of one international migrant every 46 seconds, and a net gain of one person every 17 seconds.

The United States Census also states that the population of Texas as of July 1st, 2019 is estimated to be just under 29 million consisting of 12.6% being persons 65 years and over, 25.8% being persons under the age of 18, and 7.1% being persons under the age of 5. This website does not include a percentage of people ages 18 through 65, but using arithmetic we can deduce the percentage for that age range.

First, we will add the percentage of persons 65 and over with persons under 18 years of age. However, we will not add percentage of persons under 5 years, because it is included within the persons under 18 years. Given that 25.8% and 12.6% equals 38.4%, we can subtract that from 100% and reveal that 61.6% of the population in Texas is comprised of persons between the ages of 17 and 65.

Representing Computer Code

As I have been writing this page, I have used screenshots of text within my editor to display them, but there is another way to display code with less steps. This way involves using several elements such as <code> and <pre> to wrap code that we want presented in the browser.

When we wrap our code with the <code> element, it displays the text without any formatting that may exist in the text editor. This means that the code will be displayed all on the same line, with the base readable space between each character. If any formatting is wanted, the code wrapped with the <code> element must be also wrapped in the <pre> element. Adding the <pre> element allows the text editor to differentiate between indetations, regular spaces, and enter spaces.

Below is an example of using only the <code> element followed by the same text using both the <pre> and <code>elements. Notice how the font and point size change with these added elements.

Marking up Times and Dates

Having an ability to mark dates in the browser creates more opportunity for the utilization of these dates within css or javascript. However, it is not as simple as writing today's date or the current time and wrapping it with the <time> element and being finished. Due to the variety of formats and languages that people write dates and times, the element datetime must also be utilized within the <time> element.

The datetime element works by numerically specifying the date in the order of Year-Month-DayTHour:Minute:Seconds.Miliseconds+offset of Hour:Minute.

The code would be written as such:

<time datetime="2020-06-28T17:00:00+00:00"> It is currently 5:00 PM on Sunday, June 28th, 2020.</time>

Here is the same code in practice:

It is currently 5:00 PM on Sunday, June 28th, 2020.
