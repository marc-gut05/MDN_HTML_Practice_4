Representing Computer Code

As I have been writing this page, I have used screenshots of text within my editor to display them, but there is another way to display code with less steps. This way involves using several elements such as <code> and <pre> to wrap code that we want presented in the browser.

When we wrap our code with the <code> element, it displays the text without any formatting that may exist in the text editor. This means that the code will be displayed all on the same line, with the base readable space between each character. If any formatting is wanted, the code wrapped with the <code> element must be also wrapped in the <pre> element. Adding the <pre> element allows the text editor to differentiate between indetations, regular spaces, and enter spaces.

Below is an example of using only the <code> element followed by the same text using both the <pre> and <code>elements. Notice how the font and point size change with these added elements.

Marking up Times and Dates

Having an ability to mark dates in the browser creates more opportunity for the utilization of these dates within css or javascript. However, it is not as simple as writing today's date or the current time and wrapping it with the <time> element and being finished. Due to the variety of formats and languages that people write dates and times, the element datetime must also be utilized within the <time> element.

The datetime element works by numerically specifying the date in the order of Year-Month-DayTHour:Minute:Seconds.Miliseconds+offset of Hour:Minute.

The code would be written as such:

<time datetime="2020-06-28T17:00:00+00:00"> It is currently 5:00 PM on Sunday, June 28th, 2020.</time>

Here is the same code in practice:
